# Claude Memory Compiler

## What it is
A self-evolving memory system for Claude Code, built by **Cole Medin** ([[Wiki/people/Cole-Medin]]), that applies Karpathy's raw → compile → wiki pattern to **internal** data: the transcripts of your own Claude Code sessions.

Repo: https://github.com/coleam00/claude-memory-compiler

## The idea in one line
Every Claude Code session you run gets automatically captured as a raw log. Once a day, the compiler distils those logs into updated wiki concepts and connections. Over time, Claude "remembers" what you and it have done together — without you doing any manual filing.

## Architecture
- **`raw/`** — one file per Claude session, auto-generated by hooks
- **`wiki/`** — compiled concepts, connections, index, and an `agents.md` describing the whole system
- **Three Claude Code hooks:**
  - `session-start` — loads `agents.md` and `wiki/index.md` into the fresh session context
  - `pre-compact` — summarises the recent messages before context is about to be lost
  - `session-end` — writes the summary into the daily log file
- **Daily flush** — once a day, an extraction pass reads the new logs, pulls concepts and connections, and updates the wiki
- **Compounding loop** — every answer Claude gives becomes new input that enriches the wiki, which makes the next answer smarter

## Why it matters for Niksho
Cole's project is the **closest reference implementation** to what we want for the Niksho second brain. Differences and adaptations:

| Dimension | Cole's project | Niksho's vault |
|---|---|---|
| Source of raw data | Claude Code session logs (internal) | Mixed: external sources (articles, transcripts) AND internal (decisions, efforts) |
| Compiled wiki | Concepts + connections | Same, plus authored Atlas layer (Milo-influenced) |
| Hooks | Session-start / pre-compact / session-end | Scaffolded in the architecture, not yet implemented |
| `agents.md` | Single file for the whole system | Replaced by [[AIOS/vault-map]] and [[mi]] |
| Tool | Claude Code specific | Tool-agnostic via [[claude|claude.md]] pointer to [[mi]] |

We borrow the compiler analogy, the hook pattern, and the "wiki compounds with use" insight. We replace the pure-internal focus with a hybrid that also handles external research.

## Key ideas to keep
1. **Compiler analogy** — raw → compile → wiki, the same shape as source code → compile → executable. See [[Wiki/concepts/Compiler-Analogy]].
2. **Hooks for session continuity** — the fresh-session problem is solved by loading a small context file, not by making the model memorise.
3. **Daily flush** — don't compile synchronously; batch it. Cheaper, cleaner, more reviewable.
4. **agents.md** — one canonical file that tells any coding agent how the system works. We call ours [[AIOS/vault-map]] and [[mi]].
5. **Hot cache** — the small recency buffer. See [[Wiki/techniques/Hot-Cache]].
