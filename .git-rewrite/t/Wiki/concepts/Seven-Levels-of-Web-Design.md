# The 7 Levels of Web Design with AI

## What it is

A progression framework for the quality of AI-generated web design, from generic template output (Level 1) to custom 3D/WebGL experiences (Level 7). The key insight: design quality doesn't come from a better model — it comes from how much human direction, reference material, and iteration you feed into the process.

## The levels in brief

1. **Prompt only** → generic purple gradient template. No direction = no taste.
2. **Design skills** → install skills (UI/UX Pro Max, Frontend Design). Better colour/typography but still "AI template."
3. **Visual director** → bring screenshots from Awwwards, Godly, Dribbble. AI mimics visual references better than text descriptions.
4. **The cloner** → inspect HTML/CSS/JS of admired sites. Feed source code to Claude. Understand *how* things are built, not just what they look like.
5. **Customisation** → combine component libraries (21st.dev, CodePen) with custom assets (Midjourney images, video backgrounds). Iterate 20+ times per section. Character emerges through micro-interactions and tiny details.
6. **Outside tools** → use Stitch (Google), Figma, Pencil.dev for visual iteration. Feed designs back to Claude Code. Add subtle UX details: loading hesitation, scroll progress, glassmorphism, counter animations. This is pure tinkering mode.
7. **3D/WebGL/shaders** → beyond current AI capability for prompting (as of early 2026). Custom WebGL, shader work, game-like experiences.

## The meta-lesson

Taste is not about AI. It's about articulating your taste through increasingly specific references, source code, and custom assets. Each level is really about giving the AI more precise direction — from zero (Level 1) to pixel-perfect (Level 6).
