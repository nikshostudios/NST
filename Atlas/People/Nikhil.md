# Nikhil

## Role in Niksho
Co-founder. Engineering, agents, backend, and the one who actually ships the code that runs inside ExcelTech.

## What Nikhil does
- Builds and maintains the FastAPI web app on Railway.
- Writes the agent prompts, wires up Microsoft Graph per-recruiter, manages the Supabase schema.
- Owns the Claude API integration and token budget.
- Negotiates the trade-off between "ship something that works today" and "build the thing that scales to SaaS next quarter" — usually in favour of the former.
- Direct line to Nik (father) on what's actually happening inside ExcelTech day-to-day.

## Relationship to ExcelTech
Nik — Nikhil's father — is the founder and operator of ExcelTech. This is not a neutral pilot customer. This is family, trust, and decades of relationships Nikhil grew up around. Phase 1 works because Nikhil can sit at the dinner table and ask how the day went and get a truthful answer.

This is also why [[Atlas/Business-Model/Phase-2-GTM]] works — when Nik picks up the phone to another agency owner he's known for 20 years, the conversation is different from any vendor cold call.

## Working relationship with Shoham
- Nikhil ships. Shoham designs. Both decide.
- When the right answer isn't obvious, Shoham usually has the product-sense call; Nikhil usually has the what's-actually-feasible call. The overlap is where the interesting decisions happen.

## Preferences
- Working code over perfect plans.
- Extend rather than rewrite.
- Direct feedback on prompts and agent outputs — "this sounds robotic" is more useful than "this is fine, I guess".
