# Calendar — MOC

Time-based notes. The "when" layer of the vault, as opposed to Atlas (the "what") and Efforts (the "how").

## Sub-structure

### Daily
One note per day, only on days worth writing about. Generated by [[AIOS/skills/daily-briefing]] at the start of the day and closed by [[AIOS/skills/standup]] at end of day.

File format: `Calendar/Daily/YYYY-MM-DD.md`

Structure (from the daily-briefing skill):
- Top 3 priorities
- In motion
- Hot cache summary (what the AI thinks is important right now)
- Drift check (what's slipping?)
- Open blockers
- (End of day) Standup: yesterday / today / blockers

### Weekly
A weekly review each Friday or Sunday. Optional — only when something changed.

File format: `Calendar/Weekly/YYYY-Www.md` (e.g. `2026-W15.md`)

Structure:
- What shipped
- What slipped
- What we learned
- Next week's top 3

### Quarterly
The most important Calendar note type. One per quarter. Sets the direction, names the outcomes, and gets reviewed weekly during the quarter.

File format: `Calendar/Quarterly/YYYY-Qn.md` (e.g. `2026-Q2.md`)

Structure:
- Theme (one sentence)
- Outcomes (3–5, measurable)
- Anti-goals (what we're deliberately NOT doing this quarter)
- Weekly check-in log (appended during the quarter)
- End-of-quarter reflection (filled at the end)
