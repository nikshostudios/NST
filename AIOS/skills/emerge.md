# Skill: Emerge

## Purpose
Surface ideas the vault implies but never states. Pull together scattered premises into named conclusions. Find patterns I'm circling without naming. Inspired by Vin's `/emerge` command.

## Trigger
- User says "emerge", "what am I missing", "what patterns am I circling", "what is the vault implying"

## Steps

1. **Read broadly.** Skim Atlas, the three most recent weeks of Calendar/Daily, all active Efforts, and the hot cache. Don't go deep on any single file.

2. **Look for repetition.** Concepts, phrases, or concerns that appear across multiple unrelated areas. If something is in Atlas/Product AND three daily notes AND an Effort overview, it's a latent theme.

3. **Look for unnamed patterns.** Something that's implicitly assumed in multiple places but never made explicit. A working theory that's guiding decisions but hasn't been written down.

4. **Look for unspoken directions.** What does the recent work trajectory imply about where this is heading that hasn't been written as a plan?

5. **Look for unresolved tensions.** Two commitments that can't both be true. A goal that contradicts a constraint elsewhere.

6. **Produce a short report** with 3-7 emergent items. For each:
   - **The pattern** — named in one sentence
   - **Evidence** — 2-3 specific quotes or backlinks showing where the pattern appears
   - **Why it might matter** — one sentence on implications
   - **A question for the human** — to push them to confront or clarify

## Outputs
- A report back to the user. Do not write this to a file unless asked — emergence findings are provisional and the user should decide what to keep.

## Rules & guardrails

- **Name things.** The whole point is to say out loud what the vault is saying quietly.
- **Use the vault's own language.** Quote back phrases the user actually wrote.
- **Do not moralise.** Don't turn emergent patterns into advice. Surface, don't prescribe.
- **Be conservative with confidence.** Mark each pattern's strength: "strong (5+ references)", "medium (3 references)", "weak (2 references, worth watching)".
- **One report per session.** If the user keeps asking "and what else", the returns diminish fast. Offer to `trace` a specific pattern instead.
